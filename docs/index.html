Project Abstract

Project statement: People with asthma often need to decide whether it’s safe to be active outdoors—especially during wildfire smoke and other poor-air-quality events. Our goal was to build a simple tool that combines a person’s current symptom level with local air quality data to provide clear, actionable activity guidance.

We built the Asthma Activity Advisor, a Flutter application that lets users select a symptom category (A–C), provide a location by ZIP code or city (or via device location), and fetch real-time Air Quality Index (AQI) data from the U.S. EPA’s AirNow service. The app prioritizes current observations and falls back to a daily forecast when needed. Using the AQI category and the user’s symptom level, the app generates OK or Not Recommended recommendations for light, moderate, and vigorous activity. The app supports English and Spanish and is deployed as a web app to make access and sharing easy.

Project Description

Flutter application

A single codebase targeting web (and runnable locally on other platforms during development).

AirNow AQI integration

Fetches AQI by ZIP or city (with geocoding), using observations first and forecast as a fallback.

Recommendation engine

Maps symptom level + AQI category through a slide-rule style decision matrix to generate activity guidance.

Localization

English/Spanish UI with saved language preference for consistent return visits.

How it works

1
Collect inputs
The user selects a symptom level and enters a ZIP/city (or uses device location when available).
2
Fetch AQI
The app requests AQI data from the AirNow API (current observations; forecast fallback if needed).
3
Generate guidance
AQI category + symptom level is evaluated to produce OK / Not Recommended guidance for three activity levels.
Deployment note
The AirNow API key is injected at build time for the web build. In CI, this is provided via GitHub Actions secrets.
